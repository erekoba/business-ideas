# 実現性調査

## 副業実現性

- **必要な週稼働時間**: 立ち上げ期（1〜3ヶ月目）は週8〜10時間（営業・ツール整備・提案書テンプレ作成を並走）、安定期（受注軌道後）は週5〜8時間（1件あたり10〜15時間を月2件こなす場合、週換算で6〜8時間）
- **本業との両立しやすさ**: 中
- **理由**: 作業の大半はドキュメント作成・スクリプト実行・提案書作成であり、深夜・週末に分散して対応できる。ただし、顧客との打ち合わせ（ヒアリング・提案・納品説明）は平日日中に発生する可能性があり、本業の勤務形態によっては調整が必要。クライアントが中小企業の場合、担当者との連絡が業務時間内に集中しやすい点が両立のネックになりやすい。オンライン完結・非同期コミュニケーション前提で設計できれば両立難易度は下がる。
- **立ち上げまでの期間**: サービス設計〜初受注まで**2〜3ヶ月**が現実的な目安。内訳は、ツール・スクリプト整備（2〜3週間）、LP・提案書テンプレ作成（1〜2週間）、営業活動・初回受注（1〜2ヶ月）。知人・前職ツテへのアプローチを即日開始すれば1ヶ月以内の受注も可能だが、インバウンド経由では3〜6ヶ月を要する。
- **スケールしやすいか（自動化・外注の余地）**: 部分的に可能。ディレクトリ構造の解析・命名規則チェック・レポート生成はスクリプト＋AI APIで自動化の余地が大きく、1件あたりの稼働時間を10時間→5時間以下に圧縮できる可能性がある。一方、ヒアリング・業務文脈の理解・提案の最終判断は属人性が高く、完全自動化は困難。外注については、スクリプト実行や報告書の体裁整備は切り出せるが、副業フェーズでは外注コストと管理コストが割に合わないケースが多い。スケールの本命は「ツールのSaaS化」だが、それは副業フェーズを超えた次のステージの選択肢。

## 副業月収ポテンシャル

- **月3万円は達成できるか**: はい — 隔月1件（単価8万円）の受注で達成できる。初期の顧客は知人・前職ツテから開拓できるため、ゼロから集客を構築しなくても1件目の受注は現実的に狙える。変動費・固定費を差し引いても月3万円超の手残りが確保でき、達成難易度は低い。
- **月10万円は達成できるか**: 条件付きではい — 月2件の安定受注＋提案・実行フローの標準化が揃えば達成可能。ただし、インバウンド集客（SEO・SNS）が機能するまでに3〜6ヶ月の助走が必要であり、立ち上げ直後から月10万円を狙うのは難しい。紹介の連鎖が起き始める2〜4件目以降に現実的なラインとなる。
- **月数万円達成の最大の壁**: **顧客獲得**。ディレクトリ構造の課題は「困っているが言語化できていないペイン」であり、ニーズが顕在化していない。「AI導入の前準備」という文脈で課題を再定義し、意思決定者に刺さる言葉で訴求する編集力が集客の鍵を握る。
- **副業として取り組む価値があるか**: 条件付き — コストリスクが極めて低く、1件受注で黒字化できる財務構造は副業として合理的。ただし、AIエージェント普及の初期タイミングという時間的優位性が有効なのは1〜2年程度であり、今すぐ動き始めることが前提条件となる。半年以上の助走を許容できる忍耐力があるなら取り組む価値は高い。

## 技術的実現性

- **必要なスキル・知識**:
  - ファイルシステム・クラウドストレージ（Google Drive / SharePoint等）の構造理解
  - PythonまたはGASによるディレクトリ走査・リネームスクリプトの実装
  - ChatGPT API（またはClaude API）を使った命名規則提案・構造評価プロンプトの設計
  - 中小企業の業務フロー・情報管理の基本知識（ヒアリング力）
  - 提案書・診断レポートの構成・作成力
- **現在の自分のスキルとのギャップ**: ITリテラシーが高くPythonの基礎が扱える人材であれば、ギャップは小さい。スクリプト実装経験がない場合でも、ChatGPTを活用したコード生成＋試行錯誤で2〜4週間で実用レベルに到達できる。業務コンサルティング経験があればヒアリング・提案フェーズは即戦力になる。最大のギャップになりやすいのは「Pythonスクリプト実装」と「API連携」であり、エンジニア経験がない場合は学習期間として1〜2ヶ月を見込む必要がある。
- **使用するツール・プラットフォーム**:
  - **解析・実行**: Python（os / pathlib / shutil）、Google Apps Script（Drive API）、ChatGPT API / Claude API
  - **提案・報告書**: Notion、Google Slides、Canva
  - **LP・集客**: STUDIO（無料プラン）、note、X（旧Twitter）
  - **顧客管理・連絡**: Googleフォーム、Notion、Gmail
- **学習コスト**: 中 — スクリプト実装とAPI連携の経験がある場合は低。未経験からの場合は、実用レベルまで1〜2ヶ月・週5時間程度の学習投資が必要。ただし、学習コストの大半は「ツールの使い方」ではなく「クライアントのビジネス文脈を理解しながら構造を提案する判断力」の習得にあり、これはツール学習では補えない経験値の積み上げが必要。

## リスクと対策

| リスク | 深刻度（高/中/低） | 対策 |
|--------|-------------------|------|
| 顧客の課題が潜在的すぎて集客に時間がかかる（半年以上受注ゼロの可能性） | 高 | 「AI導入準備」「ファイル整理」という顕在ニーズ寄りのキーワードで訴求し、ペインを言語化した事例記事をnote・SNSで先行発信する。知人ツテへの直接アプローチで1件目を早期に確保し、事例を作ることを最優先にする |
| ファイル移動・リネームの実行ミスによる顧客データ損失・トラブル | 高 | 実行前にフルバックアップを必須条件にする、リネーム・移動はスクリプト実行前に「変更リスト」を提案書形式で顧客確認させる、本番実行は顧客立ち会いまたは顧客自身が承認後に行う、免責事項を契約書・発注書に明記する |
| AI・SaaS企業が同様の機能を無償提供し始める（競合リスク） | 中 | 「ツール提供」ではなく「業務文脈を理解した上での設計コンサルティング」として差別化する。ツールの自動化範囲が広がっても、ヒアリング・提案・実行伴走という人的付加価値の部分で収益を確保する方向にシフトする |
| 副業の稼働時間超過による本業・生活へのしわ寄せ | 中 | 月受注上限を2件に明示的に設定する。提案書・診断フローのテンプレ化を徹底し、1件あたり稼働を10時間以内にキャップする。繁忙期は受注を断るルールを事前に決めておく |
| 単価の値下げ交渉・安く叩かれるリスク | 低 | 「AI-Ready化」という文脈で費用対効果（AI導入コスト削減・業務時間短縮）を定量的に提示する。類似のITコンサル相場（10〜30万円）を根拠に価格の正当性を説明できる資料を準備する |

## 結論

- **副業実現性**: 中
- **技術実現性**: 中
- **総評**: 財務構造・技術的難易度ともに副業として参入しやすい水準にあり、初期リスクは限りなく小さい。最大の課題は「潜在ニーズを顕在化させる集客力」であり、ここへの投資（発信・営業・事例作り）を立ち上げ初期に集中できるかどうかが成否を分ける。AIエージェント普及が加速する現在のタイミングは追い風であり、「AI導入の前準備」という文脈でポジションを取る時間的優位性は1〜2年程度と見込まれる。今すぐ動き始め、3ヶ月以内に1件の実績を作ることが、このビジネスを副業として成立させる唯一かつ最短の道筋である。
