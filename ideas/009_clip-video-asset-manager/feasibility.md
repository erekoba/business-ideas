# 実現性調査

## 副業実現性

- **必要な週稼働時間**: 立ち上げ期（1〜3ヶ月目）は週15〜20時間、安定運用期（4ヶ月目以降）は週5〜8時間
- **本業との両立しやすさ**: 中
- **理由**: 開発フェーズはまとまった集中時間が必要なため平日夜+週末の確保が必須。ただし運用フェーズはインフラ自動化・Stripe決済自動化により週次の定型作業はほぼ不要になる。ユーザーサポート（問い合わせ対応・バグ修正）が突発的に発生する点が本業との両立リスク。特にAPIの仕様変更（YouTube Data API・OpenAI）への対応は予告なく発生するため、ゼロ稼働では維持できない。
- **立ち上げまでの期間**: MVP（最低限動くベータ版）は4〜8週間。Claude Codeを活用した爆速開発を前提にしても、YouTube API連携・音声認識処理・ダッシュボードUI・Stripe決済の4点セットを品質担保しながら仕上げるには2ヶ月を見ておくのが現実的。1ヶ月は楽観的すぎるリスクがある。
- **スケールしやすいか（自動化・外注の余地）**: 中。インフラ・決済・音声処理はほぼ全自動化可能で、ユーザー数が増えても稼働時間は大きく増えない設計にできる。ただしカスタマーサポートと機能改善は自動化できず、ユーザー増に比例して負荷が増す。外注はUI改善・LP制作などは切り出しやすいが、コア機能（API連携・ロジック）は自分以外が触れない状態になりやすく、属人化リスクがある。

## 副業月収ポテンシャル

- **月3万円は達成できるか**: はい — 理由: 有料ユーザー30〜40人の確保が条件だが、切り抜き投稿者コミュニティへの直接アプローチ＋無料プランによる体験獲得を継続すれば、3〜6ヶ月での到達は現実的な範囲。損益分岐点が低い（固定費2万円程度）ため、少ない顧客数でも黒字転換できる構造は強み。
- **月10万円は達成できるか**: 難しい — 理由: 有料ユーザー67〜100人が必要だが、日本国内の本格的切り抜き投稿者は数千人規模に過ぎず、有料転換可能な層はさらに絞られる。口コミだけでは漏斗の入口が慢性的に狭く、バズや著名投稿者との提携という再現性の低いイベントに依存せざるを得ない。単一ユースケースのまま12〜18ヶ月以内に達成するのは相当な幸運が必要。
- **月数万円達成の最大の壁**: 市場の狭さ。顧客獲得コストそのものは低く抑えられるが、そもそもリーチできるターゲット総数が少ない。単価・稼働時間・技術は改善余地があるが、市場サイズだけは設計でコントロールできない構造的制約。
- **副業として取り組む価値があるか**: 条件付き。「月10万円以上を副業収入の主軸にしたい」という目的には向かない。「SaaS開発・運用スキルの実地習得＋月数万円の副収入」を目的とするなら費用対効果は高い。隣接市場（ポッドキャスト要約・企業の社内動画管理）への横展開を最初から視野に入れた設計で取り組むことが、価値を最大化する条件。

## 技術的実現性

- **必要なスキル・知識**: Webアプリ開発（フロントエンド＋バックエンド基礎）、YouTube Data API連携、音声認識API（OpenAI Whisper等）の実装、非同期処理・ジョブキュー設計（音声処理は重い）、Stripe決済組み込み、クラウドインフラ（Vercel / AWS Lambda等）の基本操作、UIダッシュボード設計
- **現在の自分のスキルとのギャップ**: Claude Codeを活用することで「書けないコードを書いてもらう」は補完できる。ただし生成されたコードのレビュー・デバッグ・API仕様の理解は自分で行う必要があり、プログラミング経験ゼロの場合は学習コストが跳ね上がる。特に非同期処理・エラーハンドリング・APIレート制限への対処は経験値が問われる領域で、ここのギャップが最もプロダクト品質に直結する。
- **使用するツール・プラットフォーム**: Claude Code（開発加速）、YouTube Data API v3、OpenAI Whisper API（音声文字起こし）、Next.js / React（ダッシュボードUI）、Vercel（ホスティング）、Supabase（DB・認証）、Stripe（決済）、BullMQ等（ジョブキュー）
- **学習コスト**: 中。個別技術は学習リソースが豊富で独学可能だが、これらを組み合わせて1つの動くプロダクトに仕上げる「統合力」に実践経験が要求される。Claude Codeがスキャフォールディングを担ってくれるため、ゼロから全部書くよりは大幅に短縮できる。

## リスクと対策

| リスク | 深刻度（高/中/低） | 対策 |
|--------|-------------------|------|
| YouTube API利用規約違反・サービス停止（動画のダウンロード・音声抽出がグレーゾーン） | 高 | 動画ダウンロードは行わず、YouTubeが公式提供するキャプション・メタデータのみを利用する設計に徹底する。音声処理はユーザーが自分で取得したファイルをアップロードする形式にしてリスクを分散。リリース前に利用規約を法的観点で再確認する |
| APIコストがユーザー増とともに利益を圧迫する（変動費の逆転現象） | 高 | プランごとに月間処理本数の上限を設定し、上限超過は次プランへのアップグレード案内に誘導。ローンチ前にユーザーあたりのAPI原価を試算してプラン単価に織り込む。処理上限設計はMVP時点から必ず実装する |
| 市場規模の天井が低く、有料ユーザー獲得が頭打ちになる | 中 | 最初からポッドキャスト・企業動画・ラジオ等の隣接市場への横展開を想定したアーキテクチャで設計。切り抜き市場で検証したコアロジックを転用できる構造にしておく |
| OpenAI等の外部API価格改定・仕様変更でコスト構造が崩れる | 中 | 音声認識はWhisperのセルフホスティング（コスト削減）や代替API（AssemblyAI等）への切り替えができるよう、APIレイヤーを抽象化して実装しておく |
| 競合（大手動画編集ツール・AI要約ツール）が同機能を追加する | 中 | 切り抜き投稿者コミュニティに特化したUX（チャット盛り上がり連携・タイムスタンプ管理）を深掘りし、汎用ツールが真似しにくい「界隈特化」の体験を武器にする。早期ユーザーとの関係を資産化する |
| 副業の稼働時間不足でサポート・改善が滞り、解約率が上がる | 低 | ユーザー向けFAQ・自己解決ドキュメントを早期に整備。バグ報告はGitHub IssuesやDiscordで非同期対応できる仕組みを整える。本業繁忙期の対応ルールをあらかじめ決めておく |

## 結論

- **副業実現性**: 中
- **技術実現性**: 中
- **総評**: 技術的には「作れないことはない」水準であり、Claude Codeを活用した開発加速を前提にすれば2ヶ月のMVP到達は現実的。副業としての最大の制約は技術ではなく市場の狭さであり、切り抜き投稿者という母数の小さいニッチに張ることで月3万円は射程に入るが、月10万円は単一市場では構造的に難しい。このアイデアを「稼ぐためだけのもの」として評価すると期待値は低めだが、「SaaS開発の実地訓練として最小リスクで走れ、かつ隣接市場への横展開の足場を作れる学習投資」として評価すると取り組む意義は十分ある。YouTube API利用規約違反とAPIコストの逆転現象という2つの高深刻リスクは、ローンチ前に必ず設計レベルで潰しておくことが継続の前提条件。
