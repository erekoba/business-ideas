# 実現性調査

## 副業実現性

- **必要な週稼働時間**: 立ち上げ期（1〜3ヶ月）：週8〜12時間 / 安定運営期（4ヶ月以降）：週4〜6時間
- **本業との両立しやすさ**: 中
- **理由**: コンテンツ制作の中核となる「ゲームプレイ＋分析＋執筆」は非同期で進められるため、深夜・休日に分散しやすい。一方、1タイトルあたりの調査に実プレイ時間（最低5〜15時間）が必要であり、これが最大のボトルネックになる。「積みゲーを資産に変える」という設計思想と相性は良いが、調査対象を新作に追いかけようとすると本業繁忙期との競合が生じやすい。初期はプレイ済みの既知タイトルから着手することで稼働時間を圧縮できる。
- **立ち上げまでの期間**: 最短4〜6週間（既プレイタイトル5本分のコンテンツ＋静的サイト＋SNSアカウント開設まで）。Steam拡張機能の実装まで含める場合は3〜4ヶ月。
- **スケールしやすいか（自動化・外注の余地）**: 中程度。チャート可視化部分はテンプレート化・フォーマット化することで制作工数を半減できる。将来的にはユーザー投稿型のデータ収集（クラウドソーシング化）に移行できれば労働依存度が下がる。ただしネタバレ管理・品質担保のためのレビュー工数は残るため、完全自動化は困難。外注ライターを使う場合も「ゲームを実際にプレイできる人材」という条件が調達コストを上げる。

## 技術的実現性

- **必要なスキル・知識**:
  - Webサイト構築（静的サイトジェネレーター or WordPressレベル）
  - データ可視化（Chart.jsやTableauPublicなどの基礎利用）
  - ブラウザ拡張開発（Chrome Extension：JavaScript基礎）※拡張なしでスタートする場合は不要
  - SEOライティングの基礎
  - Steamストアページ・API仕様の理解（Steam Web API）
  - SNS運用・コミュニティマーケティング

- **現在の自分のスキルとのギャップ**: サイト構築・ライティングのスキルがあれば最小構成で即着手可能。ブラウザ拡張開発はJavaScriptの基礎知識があれば独学3〜4週間で実装レベルに達する。データ可視化もChart.jsはドキュメントが充実しており、学習コストは低い。最大のギャップはSteam API連携の実装経験だが、公開されているサンプルコードが豊富なため深刻な障壁にはならない。
- **使用するツール・プラットフォーム**:
  - サイト構築：Astro / Next.js（静的生成）or WordPress
  - ホスティング：Vercel / Netlify（無料枠で十分）
  - 可視化：Chart.js / Recharts
  - 拡張機能：Chrome Extension Manifest V3
  - 決済：Stripe
  - CMS・執筆：Notion / Obsidian → Markdown出力
  - SNS流入：X（旧Twitter）、note、Reddit日本語圏
  - データ管理：Googleスプレッドシート（初期フェーズ）

- **学習コスト**: 中

## リスクと対策

| リスク | 深刻度（高/中/低） | 対策 |
|--------|-------------------|------|
| コンテンツ品質の担保困難：「ネタバレなし」の線引きが曖昧になり、ユーザーから批判・信頼損失が起きる | 高 | 公開前にネタバレ判定チェックリストを明文化し、全コンテンツに適用する。読者からのネタバレ報告フォームを設置し迅速に修正する運用フローを初期から設計する |
| プレイ時間の確保不足：本業繁忙・ライフイベントでコンテンツ更新が止まりSEO・読者離れが加速する | 高 | 初期30〜50タイトルはプレイ済み作品で賄い「ストック型」コンテンツを積み上げる。更新頻度の低下をカバーするため、ユーザー投稿・読者レポートの仕組みを早期に導入する |
| 競合メディアの参入・模倣：HOWLONGTOBEATやOpenCriticなど既存サービスが類似機能を追加し差別化が消失する | 中 | 「ネタバレなし」×「感情曲線の可視化」という表現軸を徹底し、データの深さよりもUX・ブランドで優位性を維持する。コミュニティ（Discord等）形成により乗り換えコストを高める |
| Steamアフィリエイト規約変更・報酬率引き下げ：収益の柱が突然削られるリスク | 中 | 収益源をSteamアフィリエイト一本に集中させず、Amazon・楽天ゲームへの分散と月額サブスクの早期育成で依存度を下げる |
| サブスク解約率の高止まり：月額300円でもゲーマー層のサブスク疲れにより早期解約が相次ぐ | 中 | 初月無料トライアルを設けてエンゲージメントを確認してから課金移行させる設計にする。解約理由アンケートを自動送信し、継続課金の価値再設計に活用する |
| ゲーム購入・プレイコストの累積：調査用ゲーム費が月1〜1.5万円に積み上がり、収益回収前に持ち出しが増大する | 低 | Steamセール・バンドル品・レビューキーの活用でコストを抑制する。初期はフリープレイ・PCゲームパス（Xbox Game Pass）収録タイトルを優先的に調査対象とする |

## 結論

- **副業実現性**: 中
- **技術実現性**: 中
- **総評**: 最小構成（静的サイト＋既プレイタイトルのコンテンツ＋SNS発信）であれば今すぐ着手でき、初期投資リスクはほぼゼロという点で副業の入り口としての合理性は高い。ただし、収益の本格化には12〜18ヶ月の継続的なコンテンツ蓄積が必要であり、「プレイ時間の確保」と「ネタバレなしという品質基準の維持」という二つの制約が常に運営の上限を規定する。技術面ではWebサイト構築と可視化の基礎スキルがあれば独力で完結できるが、ブラウザ拡張の実装は収益が立ち上がった段階で着手するフェーズ分けが現実的である。最大の価値はコンセプトの独自性にあり、early adopter層への刺さり方次第でSNSバイラルによる非線形な成長も期待できる。「まず3本書いてSNSで公開する」という検証を2週間以内に実行し、反応を見てから本格的なサイト構築に投資するかを判断するアプローチが最もリスクを抑えられる。
