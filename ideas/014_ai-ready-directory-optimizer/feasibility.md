# 実現性調査

## 副業実現性

- **必要な週稼働時間**: 立ち上げ期（初月〜2ヶ月）は週8〜10時間（ツール整備・提案資料作成・初期営業）、安定稼働後は案件ありで週5〜8時間、案件なし週は週1〜2時間（発信・営業活動のみ）
- **本業との両立しやすさ**: 高
- **理由**: 作業の大半がスクリプト実行・レポート作成・オンライン打ち合わせで構成されるため、平日夜・週末に分散して対応可能。顧客への納品期限も「1〜2週間」単位で設定できるため、急な本業の繁忙と競合しにくい。常駐・即日対応が不要なプロジェクト型の仕事であることが両立のしやすさを支えている。
- **立ち上げまでの期間**: サービス設計・ツール整備・簡易LP公開まで**4〜6週間**。人脈経由の初案件獲得を含めると**2〜3ヶ月**を現実的な目安とする。
- **スケールしやすいか（自動化・外注の余地）**: 中。診断スクリプト・命名規則テンプレート・提案書フォーマットの整備により、1件あたりの工数は初回20〜30時間から2〜3件目以降は10〜15時間へ圧縮可能。完全自動化は困難（顧客のコンテキスト理解・ヒアリングが必須）だが、スクリプト実行・レポート生成の定型部分は外注（クラウドワークス等）への切り出しが中期的に可能。ただし副業フェーズでは外注管理コストが逆効果になるリスクがあるため、月3件超えを目安に検討するのが現実的。

## 副業月収ポテンシャル

- **月3万円は達成できるか**: はい — 年4〜5件（＝2〜3ヶ月に1件）の受注で到達可能。既存の業務人脈へのアプローチのみで初案件を獲得できる蓋然性が高く、特別な集客投資を必要としない。ハードルは低い。
- **月10万円は達成できるか**: はい（ただし3〜6ヶ月の助走期間を要する）— 月1件の単発案件（20万円）＋サブスク転換2〜3社（月4〜6万円）の組み合わせで到達可能。SNS・noteでの継続的な情報発信による認知獲得と、初期顧客のサブスク転換率が鍵を握る。月10〜15時間の稼働確保が前提条件。
- **月数万円達成の最大の壁**: 顧客獲得。「AIのためにファイル整理に予算を使う」という発想が顧客企業に存在しないため、予算カテゴリの創出と概念説明に要するコストが最大の障壁。単価・技術・稼働時間はいずれも問題にならない水準であり、唯一の制約は案件の発見と成約である。
- **副業として取り組む価値があるか**: 条件付き — 「AI活用・DX推進に関わる職種や人脈をすでに持っている人」にとっては費用対効果が極めて高い副業。一方、IT・業務改善領域の人脈がゼロの場合は初案件獲得に6ヶ月以上かかるリスクがあり、その場合は発信活動との並走が必須条件となる。

## 技術的実現性

- **必要なスキル・知識**:
  - PythonまたはGAS（Google Apps Script）によるファイルシステム操作スクリプトの作成
  - Google Drive / Windows共有フォルダの構造取得・一括リネーム処理の実装
  - LLM API（OpenAI等）を用いた命名規則の分析・提案ロジックの構築
  - ディレクトリ設計・情報アーキテクチャの基礎知識
  - 提案書・変更レポートの作成能力（ビジネス文書レベル）
- **現在の自分のスキルとのギャップ**: Pythonの基礎とAPI連携経験があれば技術的ギャップはほぼない。GASはGoogle Workspace案件に限定されるが学習コストは低い。ディレクトリ設計の体系的な知識は独学・実案件での経験で補完可能。最大のギャップは「技術」ではなく「業務コンテキストの理解と提案力」であり、これはヒアリング設計とテンプレートの蓄積で解決できる。
- **使用するツール・プラットフォーム**:
  - 開発：Python（osモジュール・OpenAI API）/ GAS
  - 顧客環境：Google Drive API / Windows SMB共有フォルダ
  - 提案・納品：Notion / Google Docs / Googleスプレッドシート
  - LP・集客：STUDIO または Notion（無料〜月1,000円）
  - コミュニケーション：Google Meet / Zoom（無料枠で対応可）
- **学習コスト**: 低（Python・API連携の基礎があれば追加学習は10〜20時間以内で完結する見込み）

## リスクと対策

| リスク | 深刻度（高/中/低） | 対策 |
|--------|-------------------|------|
| 「AIのためのファイル整理」に予算を出す企業が少なく、初案件獲得に時間がかかる | 高 | 「AI導入準備」ではなく「業務効率化・検索時間の削減」という既存の課題解決として訴求し、AI文脈は副次的メリットとして提示する。既存人脈への直接アプローチを優先し、SNS発信は中期施策と割り切る |
| 顧客環境（オンプレサーバ・独自権限設定）へのアクセス制限によりスクリプト実行が困難になる | 中 | 初期ヒアリングで環境条件を必ず確認し、アクセス不可の場合は「診断レポートのみ」プランへ誘導する。顧客側担当者にスクリプトを渡してセルフ実行してもらう手順書の整備もオプションとして準備する |
| ファイルの大量リネーム・移動による誤削除・業務停止リスク | 中 | 実行前に必ずバックアップ確認を契約条件に明記し、本番実行前にドライラン（変更内容のプレビュー出力）を必須プロセスとして組み込む。免責事項を契約書に記載する |
| 競合（大手SIer・コンサル）が同様のサービスをパッケージ化して市場参入する | 低 | 副業フェーズでは中小企業・スタートアップに絞ることで大手との競合を回避。スピード・低価格・担当者との直接コミュニケーションを差別化軸とする |
| サブスク顧客の解約率が高く、安定収入の土台が作れない | 中 | 月次保守の価値を「新規フォルダ設計の相談窓口」として具体化し、解約されにくい運用設計にする。最低3ヶ月契約を初期条件として設定する |

## 結論

- **副業実現性**: 高
- **技術実現性**: 高
- **総評**: 技術的ハードルが低く初期コストがほぼゼロであるため、副業ビジネスとしての参入障壁は極めて低い。収益性調査で示された通り粗利率95%超・1件目から黒字という構造は副業として理想的であり、週5〜10時間の稼働で月10〜40万円のゾーンが現実的に射程に入る。最大のリスクは一貫して「顧客獲得の難度」であり、これは技術や収益の問題ではなく純粋に営業・発信の問題である。裏を返せば、既存の人脈や発信力を持つ人間が参入した場合の有利さが際立つビジネスでもある。まず無料ヒアリングとプロトタイプ作成を2週間以内に着手し、仮説を実案件で検証することが最優先事項である。
