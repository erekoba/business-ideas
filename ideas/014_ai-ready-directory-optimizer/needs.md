# ニーズ調査

## 市場規模

- **市場規模の概算**:
  潜在顧客は「Google WorkspaceまたはMicrosoft365を導入済みで、AIエージェント導入を検討している中小〜中堅企業」。
  国内のGoogle Workspace契約企業は約60万社（2024年推定）、Microsoft365は約200万社超。このうちAIエージェント導入を検討・計画中の企業を5〜10%と仮定すると、**潜在ターゲットは約13〜26万社**。
  単価を診断＋実行パッケージ30万円/件、年間受注可能社数を市場の0.1%（130〜260社）と保守的に見積もると、**市場全体の取り得る売上規模は約39〜78億円/年**。
  副業単独での取り分は年間数百万円規模が現実的な上限。

- **成長トレンド**:
  拡大中。2024〜2025年にかけてCopilot for Microsoft365・Gemini for Google Workspaceの普及が加速しており、「AIを導入したが使えない」という失敗事例が増加。ファイル整備への需要は今後2〜3年で急増すると見られる。AIエージェント市場自体が国内でも年率30%超の成長予測（IDC等）であり、本サービスはその前処理需要として連動して伸びる。

- **根拠・参考データ**:
  - Google Workspace国内契約社数：Google公式発表・ITmedia報道（2023〜2024年）
  - Microsoft365国内利用状況：MM総研（2024年）
  - 国内AIエージェント市場規模：IDC Japan「国内AI市場予測（2024年）」
  - 中小企業のDX推進状況：中小企業庁「2023年版中小企業白書」（DX取組率約30%、うち実行フェーズは少数）

## 顧客ニーズの検証

- **顧客が今どう解決しているか（代替手段）**:
  - **放置**: 最多パターン。問題認識はあるが工数・優先度の問題で手がつかない
  - **フリーランスSEへの依頼**: 口コミ経由で単発受託。AI観点はなくファイル移動・整理に留まる
  - **社内の担当者が手作業**: 総務・情シスが時間を捻出して対応。属人的かつ非効率
  - **ChatGPT/ClaudeをDIY活用**: ITリテラシーの高い一部担当者が命名規則をプロンプトで相談するが、実行（リネーム）は手作業

- **不満・ペインポイント**:
  - ファイル名・フォルダ名がバラバラで検索ヒット率が低く、業務時間のロスが慢性的に発生している
  - AIツールを導入しても「ファイルを見つけられない・読み間違える」という問題が解消されず、ROIが出ない
  - 整理ルールを決めても社内に浸透せず、すぐに元の状態に戻る（ルール形骸化）
  - 手作業でのリネーム・移動は数百〜数千ファイル規模になると現実的に不可能
  - 「何から手をつければいいか分からない」という構造的な課題の見えにくさ

- **支払い意欲があるか**:
  **中〜高**。ただし条件付き。
  「AIエージェント導入」という上位目的がある場合、その投資（月額数万〜数十万円）の前処理として10〜30万円は正当化しやすい。一方、ファイル整理単体での予算計上は稟議が通りにくい中小企業も多く、「AI導入準備費」としての予算付けが訴求の鍵となる。経営者直決裁が通りやすい50万円以下の価格設定が有効。

## 競合分析

| 競合名 | 強み | 弱み |
|--------|------|------|
| IT導入支援コンサル（アクセンチュア等） | ブランド信頼性・総合DX支援力・大企業との取引実績 | 中小企業に届かない高単価・ファイル整理は主業務外・AI-Ready観点の訴求なし |
| フリーランスSE（Workspace整理） | 低コスト・柔軟対応・現場の温度感を理解 | AI観点が薄い・スケールしない・属人的で品質にばらつき |
| Avepoint等ガバナンスSaaS | 権限管理・ガバナンス機能が強力・Microsoft365との親和性 | 命名規則の提案機能なし・日本語対応が弱い・AI-Ready化は対象外 |
| NotePM・Confluence等ナレッジSaaS | ナレッジ管理として定着・UIが使いやすい | 既存ファイルサーバの構造変換は対象外・導入後の新規情報管理が主目的 |
| ChatGPT/Claude（DIY） | 低コスト・即時利用・提案の質は高い | 実行（リネーム・移動）は手作業・プロンプト設計スキルが必要・再現性がない |

## 差別化ポイント

**① 「AI-Ready化」という文脈での唯一の訴求**
既存競合はファイル整理をIT作業として捉えているが、本サービスは「AIエージェントが機能するための前処理」として位置づける。この切り口での競合は現時点で国内に存在せず、訴求メッセージの独占状態にある。

**② 診断〜提案〜実行までの一気通貫**
フリーランスSEは依頼内容の実行に留まり、ChatGPT DIYは提案止まり。本サービスは「現状診断 → AI-Ready命名規則の設計 → スクリプトによる自動リネーム実行 → 変更レポートの納品」までをワンパッケージで提供し、顧客の工数をほぼゼロにする。

**③ 中小〜中堅企業に刺さる価格帯と意思決定スピード**
大手コンサルが入らない100名以下の企業を主戦場とし、10〜30万円という経営者が即決しやすい価格設定で、提案から着手まで最短1〜2週間を実現する。

**④ 将来のSaaS化・横展開への布石**
受託で蓄積した業種別・規模別の命名規則テンプレートをプロダクト化することで、スケーラブルなビジネスへの転換パスを持つ。競合SaaSが持たない「AI可読性スコア」などの独自指標を設計することで、プロダクト上の参入障壁を先行して構築できる。

## 結論

- **ニーズの強さ**: 高
- **理由**:
  「AIツールを導入したのに使えない」という失敗体験が2024〜2025年にかけて急増しており、その根本原因がファイル構造の混乱にあるという認識が現場レベルで広がり始めている。ペインは慢性的かつ深刻で、現状の代替手段（放置・手作業・フリーランスSE）はいずれも根本解決になっておらず、明確な「解決済み市場」が存在しない。AIエージェント導入投資の正当化コストとして本サービスの費用を組み込めるため、支払い意欲も引き出しやすい構造にある。競合が手薄な中小企業市場における空白地帯に位置しており、今参入することで認知と実績の先行者優位を確立できる可能性が高い。
