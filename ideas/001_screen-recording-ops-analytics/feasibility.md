# 実現性調査

## 副業実現性

- **必要な週稼働時間**: 
  - 営業・提案：週2〜4時間
  - 案件実施中（録画設定〜解析〜レポート作成）：週5〜10時間（案件中の2〜3週間）
  - **平均：週5〜8時間**（案件がないタイミングは週2〜3時間）

- **本業との両立しやすさ**: 中
- **理由**: 
  - 顧客先PCへのエージェント設定は平日日中が必要なケースあり（リモート設定なら時間外対応可）
  - 解析・レポート作成は夜間・週末で対応可能
  - 月1件ペースなら十分両立可能。月2件は繁忙期と重なると厳しい

- **立ち上げまでの期間**: 
  - **最短1ヶ月**：既存のOBS Studio（画面録画）＋手動解析＋Notionレポートで初案件受注可能
  - **3ヶ月**：録画エージェント（簡易版）＋AI解析スクリプト＋ダッシュボード（Looker Studio）で準プロダクト化
  - **6〜12ヶ月**：フル機能SaaSプロトタイプ

- **スケールしやすいか（自動化・外注の余地）**: 
  - AI解析パイプラインは自動化余地が大きい（反復作業をスクリプト化）
  - レポートの定型部分はAIに下書き生成させられる
  - 解析アシスタントのクラウドソーシング活用（ランサーズ等）も可
  - 営業は最初のうちは自分でやる必要がある

## 技術的実現性

- **必要なスキル・知識**: 
  - PC画面録画ツールの設定・運用（OBS, PowerAutomate等）
  - Python（OCR処理・ログ解析スクリプト）
  - AI API活用（OpenAI GPT-4o Vision, Google Cloud Vision API）
  - データ可視化（Looker Studio, Tableau Public等）
  - プライバシー・労働法規の基礎知識（重要）

- **現在の自分のスキルとのギャップ**: 
  - Pythonとデータ分析の基礎があれば技術的ハードルは低い
  - OCR・画像認識はAPIで解決できるため自前実装不要
  - 最大のギャップは「法務・プライバシー設計」と「営業・顧客開拓」

- **使用するツール・プラットフォーム**: 
  - 録画：OBS Studio（無料）またはPowerAutomate Desktop（Windows標準）
  - 解析：Python + OpenAI Vision API / Google Cloud Vision
  - ストレージ：AWS S3 or Google Drive（初期）
  - レポート：Looker Studio（無料）or Notion
  - 請求・契約：freee / ミスミのクラウドサイン

- **学習コスト**: 低〜中
  - 録画・基本解析：低（既存ツールで対応可）
  - AI API連携：中（Pythonの基礎があれば1〜2週間）
  - 法務・プライバシー：中（社労士・弁護士への相談1〜2回で補完可）

## リスクと対策

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| プライバシー・労働問題（従業員監視として問題化） | 高 | 導入前の従業員への説明・同意取得プロセスを仕組み化。「改善目的」の明示と録画データの利用制限を契約に明記。社労士・弁護士監修の利用規約を整備。 |
| 録画データの情報漏洩・セキュリティ事故 | 高 | エンドツーエンド暗号化・アクセスログ管理・ISO27001準拠のクラウド利用。契約でデータ取扱責任を明確化。 |
| 顧客開拓が難しい（BtoB営業の壁） | 高 | 最初の1〜2件は知人・前職コネクションで受注。実績ケーススタディを作り、LinkedInやBizreach経由で展開。 |
| 海外製品（Teramind等）の日本語化・低価格化 | 中 | 日本業務特化の解析精度・国内データ保存・手厚いサポートで差別化。早期に顧客基盤を構築。 |
| 副業禁止規定に抵触（受注者自身のリスク） | 中 | 就業規則を確認。法人設立（合同会社）や家族名義での運営も選択肢。 |
| 案件が散発的で収入が不安定 | 中 | 最初からリテイナー契約（月次サポート料）をセットで提案し、継続収益を確保。 |
| 録画に対する現場従業員の心理的抵抗 | 中 | UIを「監視」ではなく「業務日誌の自動化」として設計。従業員本人も自分の録画を見られる仕様にする。 |
