# ニーズ調査

## 市場規模

- **市場規模の概算**: 日本のゲーム市場全体は約2兆円（2023年、CESA調べ）。そのうちPCゲーム市場は約2,000〜3,000億円規模と推定。本サービスの直接ターゲットは「Steam利用者で購入前に情報収集する習慣があるPCゲーマー」に絞られ、国内Steamアクティブユーザー数は推定300〜500万人。このうちレビューや攻略情報を能動的に調べる層を30%と仮定すると、潜在ユーザー数は約90〜150万人規模。
- **成長トレンド**: 緩やかに拡大。インディーゲームの増加（Steamの年間新作数は毎年過去最高を更新）により「どれを選ぶか」の情報収集ニーズは増加傾向。一方でPCゲーム人口自体の急拡大は見込みにくく、既存プレイヤーの深耕が主戦場になる。
- **根拠・参考データ**: CESA「2023年ゲーム白書」、Steam Spy推定データ、Valve社の公式発表（2023年のSteamアクティブユーザー数は全世界1.3億人）、国内比率を約3〜4%と仮定して算出。

## 顧客ニーズの検証

- **顧客が今どう解決しているか（代替手段）**: ①Steamのレビュー・プレイ時間中央値を確認する、②YouTubeで実況動画を視聴する（30分〜1時間以上かける場合も）、③X（旧Twitter）でゲーム名を検索して感想を集める、④友人・知人に聞く、⑤Game PassやPS Plusで試しにプレイする、⑥How Long To Beatでボリュームを確認してから判断する、といった複数手段の組み合わせが現状。
- **不満・ペインポイント**: 「レビューは人によって基準がバラバラで自分に当てはまるか分からない」「実況動画は長くてネタバレもある」「Steamの総合評価が高くても自分には刺さらなかった経験がある」「インディーゲームは情報が少なすぎて判断材料に乏しい」「結局買って数時間で積みゲーになるリスクを毎回抱えている」。特に「自分の好みとの一致度」を事前に知る手段が存在しないことへの不満は潜在的に強い。
- **支払い意欲があるか**: 低〜中。ゲーム情報収集ツール自体に課金する文化は日本では未成熟。ただし「1本のゲームが3,000〜8,000円するため、1本でも無駄買いを防げれば元が取れる」というロジックは刺さりやすく、月額300〜500円程度のサブスクなら受け入れられる可能性はある。初期はフリーミアムで習慣化させてから課金転換を狙う設計が現実的。

## 競合分析

| 競合名 | 強み | 弱み |
|--------|------|------|
| Steam（レビュー・プレイ時間表示） | プレイ時間中央値・レビュー数が膨大で信頼性が高い。購入導線と一体化しており離脱しにくい | 「自分がハマれるか」の個人適合予測は不可。レビューは主観的で玉石混交 |
| How Long To Beat | クリア時間データが世界最大規模。無料で使いやすい | プレイスタイルの個人差を考慮しない。日本語対応が弱くインディー情報も薄い |
| YouTube実況（はじめしゃちょー等） | 実際のゲーム体験を疑似体験できる。エンタメとして成立しており離脱動機が生まれにくい | 尺が長くネタバレリスクがある。「自分に合うか」の判断は最終的に視聴者任せ |
| Game Pass / PS Plus | 実際にプレイして自分で判断できる。コスパが高い | 全タイトル非対応。インディー・ニッチ作品は少なく、PCゲームは対応外も多い |
| Metacritic / OpenCritic | メディアレビューの集積で権威性がある | 没入感・中毒性の予測には不向き。ユーザーの個人趣味に非対応 |

## 差別化ポイント

既存競合はすべて「大衆の平均値」を提供しており、「あなた個人がこのゲームにハマれるか」を正面から解決しているサービスは現時点で存在しない。本サービスは以下3点で差別化できる。

**①個人プロファイルとの適合スコア化**: プレイ履歴・好みジャンル・感情的反応傾向（ストーリー重視か爽快感重視かなど）をアンケートで取得し、ゲームのタグ情報（Steam API・IGDB API）と照合して「あなた専用の没入スコア」を算出する。平均点ではなく個人差を起点にする点が最大の独自性。

**②日本語完全対応・国内インディーゲームの網羅**: 英語サービスが手薄な国内インディー・同人ゲーム市場（BOOTH・DLsiteなど）も対象とすることで、英語圏競合が入り込めないニッチ市場を先行確保する。

**③SNSバイラル設計**: 「私のハマり予測スコア●点→実際に●時間プレイしました」といった事後報告の共有機能を設け、診断結果のシェアをエンゲージメントとマーケティングに同時活用する。コスト0で口コミ拡散を狙える設計。

## 結論

- **ニーズの強さ**: 中
- **理由**: 「購入前にハマれるか知りたい」というペインポイントは実在し、積みゲー問題として多くのゲーマーが体感している普遍的な課題である。ただし現状は無料の代替手段を組み合わせることで「それなりに解決している」ため、新サービスへ乗り換える動機は強くない。ニーズの存在は確かだが「今すぐ課金してでも解決したい切実な痛み」かどうかは疑問が残る。また予測精度がサービスの生命線であり、データが少ない初期段階ではユーザーの信頼を獲得しにくいコールドスタート問題が最大の参入リスクとなる。副業規模でのMVP検証（Googleフォーム×スプレッドシートで診断ロジックを手動運用するなど）から始め、「予測スコアと実際の体験が一致した」という成功体験を積み重ねて口コミを作ることが、最初の90日間で最優先すべき行動となる。
