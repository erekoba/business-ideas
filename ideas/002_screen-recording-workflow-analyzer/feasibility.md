# 実現性調査

## 副業実現性

- **必要な週稼働時間**: 
  - **MVP・検証フェーズ（最初の3ヶ月）**: 週15〜20時間（ツール設定・初回顧客対応・レポート作成）
  - **安定稼働フェーズ**: 週8〜12時間（案件対応・改善・営業）
  - ※ 解析の自動化が進めば週5〜8時間まで圧縮可能

- **本業との両立しやすさ**: 中
- **理由**: 
  - 顧客企業での録画設定・説明は平日日中が必要な場合がある（→ コンサルタント経由なら代理対応可能）
  - レポート作成・AI解析は夜間・週末でも実施可能
  - 初期顧客獲得の商談が平日に集中するため、本業の状況次第で両立難度が変わる

- **立ち上げまでの期間**: 
  - **フェーズ1（0〜1ヶ月）**: 既存ツール（OBS Studio録画 + GPT-4 Vision + Notion）を組み合わせたMVPを構築
  - **フェーズ2（1〜3ヶ月）**: 知人コンサルタント・知人企業1社で無償トライアル実施→フィードバック収集
  - **フェーズ3（3〜6ヶ月）**: 有償案件2〜3件獲得。スポット型サービスとして確立
  - **フェーズ4（6〜18ヶ月）**: エージェント開発・SaaS化・リセラー開拓

- **スケールしやすいか（自動化・外注の余地）**: 
  - **自動化できる部分**: 録画収集・OCR処理・LLM解析・ダッシュボード生成（高い自動化余地）
  - **人手が必要な部分**: 初回設定・顧客へのレポート説明・改善提案の文脈読み（一定の人手は残る）
  - **外注可能な部分**: エージェント開発（クラウドワークス等）、レポートデザイン、初期営業

## 技術的実現性

- **必要なスキル・知識**: 
  - Pythonスクリプト（画面録画・自動化）
  - クラウドサービス（AWS S3 / GCP Storage）の基礎操作
  - OCR API（Google Vision API / Azure AI Vision）の利用
  - LLM API（OpenAI GPT-4 Vision等）のプロンプトエンジニアリング
  - 基礎的なデータ可視化（Looker Studio / Tableau Publicで代用可）
  - プライバシー・労働法規の理解（個人情報保護法・不正競争防止法）

- **現在の自分のスキルとのギャップ**: 
  - IT/エンジニアリングバックグラウンドがあれば：ギャップ小（1〜2ヶ月で実装可能）
  - コンサルタントバックグラウンドがあれば：開発部分は外注が前提。ギャップ中程度
  - 非IT職の場合：ギャップ大。ノーコードツール（Zapier + Make + Notion）での代替構成を検討

- **使用するツール・プラットフォーム**: 
  - **録画**: OBS Studio（無料）or カスタムPythonエージェント（mss + schedule）
  - **ストレージ**: AWS S3 / Google Drive（初期）
  - **OCR・画像解析**: Google Vision API / OpenAI GPT-4o Vision
  - **レポート**: Looker Studio / Notion / Google Slides（初期はこれで十分）
  - **決済**: Stripe
  - **将来のSaaS化**: Electron（デスクトップエージェント）+ Next.js + Supabase

- **学習コスト**: 中（ITバックグラウンドがある場合は低）

## リスクと対策

| リスク | 深刻度（高/中/低） | 対策 |
|--------|-------------------|------|
| **従業員のプライバシー・監視への反発**（最大リスク） | 高 | 「改善PJ期間限定・本人への開示・会社との三者合意」を契約で明文化。個人識別は最小限に設計。労働組合・就業規則との整合確認を導入プロセスに組み込む |
| **個人情報保護法・労働法への抵触懸念** | 高 | 弁護士レビューの利用規約整備（10〜30万円）。本人同意取得フロー、データ保管期間の制限（PJ終了後90日で削除等）を標準化 |
| **競合（ActivTrak等）の日本語対応強化** | 中 | 「業務改善」特化のユースケース設計・日本語業務アプリ対応・コンサル向けB2B2Bで差別化。価格でも対抗可能 |
| **録画データ量が大きくインフラコストが膨らむ** | 中 | 動画ではなく定期スクリーンショット（30秒〜1分間隔）方式に変更でストレージ1/100以下に削減可能 |
| **顧客企業のセキュリティポリシーでエージェントインストール不可** | 中 | インストール不要のブラウザ拡張版（Webアプリ業務限定）を代替として用意。IT管理者向けのホワイトリスト申請サポートを提供 |
| **副業として時間が足りず品質が落ちる** | 中 | 最初の3〜6ヶ月は同時進行案件を2件以下に制限。早期に解析の自動化を優先実装 |
| **LLM解析の精度が低く誤った業務分析になる** | 低〜中 | 初期はLLM出力をコンサルタント（自分）がレビューする人間-in-the-loop方式。精度の低い箇所は「要確認」フラグを立てる |

## 結論

- **副業実現性**: 高
- **技術実現性**: 高
- **総評**: 技術的には既存のOCR・LLM・録画ツールを組み合わせるだけでMVPが作れるため、エンジニアリングの障壁は低い。最大の障壁はプライバシー・法的リスクの設計であり、ここを最初に固めることが成功の鍵。副業としては「コンサルタント向けスポット案件」から始めるのが最も現実的で、稼働時間・収益・リスクのバランスが良い。
