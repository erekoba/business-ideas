# ニーズ調査

## 市場規模

- **市場規模の概算**: 
  - **直接市場（業務プロセス改善ツール・BPM市場）**: 国内BPM/業務可視化ソフトウェア市場は約500〜700億円規模（IDC Japan推計ベース）
  - **隣接市場（業務コンサルティング）**: 国内経営・業務コンサルティング市場は約1.2兆円。うち現状分析フェーズが占めるコストは20〜30%と推定で約2,400〜3,600億円
  - **SaaS換算ターゲット**: 従業員50〜500名の中小中堅企業は国内約18万社。うち業務改善PJを年1件以上実施する企業を10%と仮定すると約1.8万社がターゲット。月額10〜30万円課金なら最大市場規模540億円/年
  
- **成長トレンド**: 
  - DX推進（政府のデジタル田園都市国家構想）により中小企業の業務改善投資は増加傾向
  - リモートワーク普及により「見えない業務」の可視化ニーズが高まっている
  - AI/LLMコストの低下により、解析精度向上と価格競争力が両立しやすくなっている
  - RPAツール導入前のアセスメント需要も拡大中

- **根拠・参考データ**: 
  - 経済産業省「DXレポート2.2」（2022）：企業の約8割がDX推進を経営課題と認識
  - 矢野経済研究所「BPM市場動向調査」：国内BPM市場は年率5〜8%成長
  - 総務省「情報通信白書」：テレワーク導入企業でのマネジメント課題として「業務実態把握困難」が上位

## 顧客ニーズの検証

- **顧客が今どう解決しているか（代替手段）**: 
  1. **ヒアリング・インタビュー**：業務担当者に「どんな業務をどのくらいやっているか」を聞く。最も一般的だが主観バイアスが大きい
  2. **業務日報・タイムシート記録**：日報や工数管理ツール（Toggl、Jootoなど）を使わせる。記録漏れ・意図的な修正が多発
  3. **マニュアル・フロー図の読み込み**：AS-ISフロー図が存在する場合のみ有効。現場実態との乖離が大きい
  4. **コンサルタントの現地観察（張り付き）**：最も精度が高いが、1人1〜2日で10〜30万円のコストがかかり、スケールしない
  5. **既存システムのログ分析**：ERPやCRMのログを見る。PCフル操作の一部しかカバーできない

- **不満・ペインポイント**: 
  - ヒアリングは「あるべき姿」が語られ、「実際の姿」が出てこない（担当者も無意識に美化する）
  - 複数人を調査するとコストが線形に増加する
  - 一時的な観察では「繁忙期」「月末処理」など特定タイミングの業務が取れない
  - エクセル職人の「神ファイル」操作など、ツール外の作業が全く見えない
  - 改善施策実施後の効果測定手段がなく、「やりっぱなし」になりやすい

- **支払い意欲があるか**: 
  - **業務コンサルタント**: 高い。現状分析コストを顧客に転嫁できれば実質負担ゼロ。むしろ提案の差別化ポイントになる
  - **社内IT・経営企画**: 中程度。予算化のハードルがあるが、年1〜2回のPJでROIを示せれば通りやすい
  - **中小企業経営者**: 低〜中。プライバシー懸念と費用対効果への疑問が先に立つ。導入事例と試算が必要

## 競合分析

| 競合名 | 強み | 弱み |
|--------|------|------|
| ActivTrak（米国SaaS） | 従業員生産性監視の老舗。豊富な実績と機能 | 英語UI中心。「監視」色が強くプライバシー問題になりやすい。日本市場対応が薄い |
| Teramind（米国SaaS） | 詳細なユーザー行動分析・インサイダーリスク検知 | 高価格（中小企業には重い）。セキュリティ監視色が強く業務改善用途に使いにくい |
| Time Doctor | タイムトラッキング＋スクリーンショット機能あり | 「監視ツール」として従業員の反発を招きやすい。AI解析機能は限定的 |
| Microsoft Viva Insights | Microsoft 365連携で導入障壁低い | Outlookなどのアクティビティのみ。PC画面全体の操作は取れない |
| 国内：GrowthBoard / KAIZEN Platform | 国内企業向けUI改善・業務分析 | 対象がWebアプリ操作に限定。デスクトップ業務には非対応 |
| 手作業コンサルティング（代替手段） | 現場担当者との信頼構築が同時にできる | コスト高・スケールしない・観察者効果あり |

## 差別化ポイント

1. **「業務改善」文脈での設計**: 競合の多くは「従業員監視・セキュリティ」文脈。本サービスは「改善のための可視化」として設計し、従業員の反発を低減する（データは改善PJ期間限定、本人にも開示など）
2. **日本語業務特化のAI解析**: 日本語OCR精度・日本語業務アプリ（弥生会計、勘定奉行、kintone等）の認識・日本語レポート生成に特化
3. **コンサルタント向けB2B2B モデル**: エンドユーザー企業への直販ではなく、コンサルタント・SIerが顧客企業に導入するホワイトラベル型。営業コストを大幅削減
4. **期間限定・プロジェクト型課金**: 常時監視ではなく「改善PJ期間中の2〜4週間だけ」という使い方を正面から設計。プライバシー懸念と価格ハードルを同時に下げる

## 結論

- **ニーズの強さ**: 高
- **理由**: 業務改善の現状分析フェーズは全企業共通の課題であり、既存手段への不満は明確。DX推進の追い風もある。ただし「従業員監視」への心理的抵抗をどう設計で乗り越えるかが普及の鍵。
